# Контекст проекта для Claude

## Что это
Учебный проект автотестов API на Python + pytest против публичного CoinGecko API v3.
Цель — резюме, поэтому код должен выглядеть как живой человеческий проект, не как enterprise.

## Стек
- Python 3.11+
- pytest + pytest-xdist + pytest-rerunfailures + python-dotenv
- requests (без httpx, без aiohttp — намеренно)

## Структура
- `utils/config.py` — загрузка .env.{ENV}, единая точка входа для всех настроек
- `utils/api_client.py` — тонкая обёртка над requests.Session
- `tests/conftest.py` — фикстуры, логирование запросов при падении, заголовок сессии
- `tests/test_ping.py` — GET /ping
- `tests/test_markets.py` — GET /coins/markets
- `tests/test_coin.py` — GET /coins/{id}

## Запуск
```bash
# локально (dev по умолчанию)
pytest -m fast
pytest -m fast -n auto
pytest -n auto

# явное окружение
ENV=prod pytest -m fast
```

## Маркеры
- `fast` — один запрос, гоняем на каждый коммит
- `slow` — параметризованные или несколько запросов, раз в день в CI

## Договорённости по коду
- Комментарии и сообщения об ошибках — на русском
- Стиль комментариев — по-человечески, не как документация к библиотеке
- Не писать docstring там где название функции говорит само за себя
- `request.node.last_response = response` — так тесты подключаются к логированию при падении

## Решения которые уже приняты — не предлагать менять
- Нет `jsonschema` — ручные assert-ы читабельнее для этого размера
- Нет моков — проект тестирует реальный внешний API, моки противоречат идее
- Нет Docker — избыточно
- Нет `httpx` или `aiohttp` — синхронный `requests` достаточен и проще

## Ограничения CoinGecko бесплатного тира
- ~10-30 запросов в минуту
- `--reruns-delay=2` в pytest.ini — не убирать, иначе при параллельном запуске
  прилетит волна 429
- При добавлении новых параметризованных тестов думать о количестве запросов

## Окружения
- `.env.dev` и `.env.prod` не коммитятся (в .gitignore)
- Шаблон — `.env.example`
- `ENV=dev` по умолчанию если переменная не задана
- В CI выставляется `ENV=prod` через `env:` блок в workflow

## Что сейчас не покрыто тестами
- `/simple/price` — простой эндпоинт для получения цены по id
- `/coins/{id}/history` — исторические данные
- `/coins/{id}/market_chart` — данные для графиков
- Негативные сценарии для /markets (неверная валюта, отрицательный per_page)
